## Identity Model

| Option | Description | Selected |
|--------|-------------|----------|
| Global user + join table | One users row per email (globally unique). tenant_memberships links users to tenants. Remove tenant_id FK from users. | ✓ |
| Per-tenant user rows (keep current) | Keep one users row per (tenant, email). A linking table ties same email across tenants. | |

**User's choice:** Global user + join table
**Notes:** Dev environment — no production data, no migration conflict. User explicitly requested editing original migrations in-place rather than adding new ones.

## Tenant Geographic Boundaries

| Option | Description | Selected |
|--------|-------------|----------|

**User's choice:** Polygons stored in DB (Recommended)
**Notes:** Add boundary column to tenants table in this phase as the foundation for Phase 3's GPS-based tenant detection.

---

## RLS on Users Table

**User's clarification:** RLS should stay as the security mechanism for tenant data isolation, but it belongs on data tables (reports, incidents), not on the users table itself. Managers need to be isolated to their tenant's data.

**Resolution:** Remove RLS from users table. Keep RLS on tenant-scoped data tables.

## Membership Role Design (evolved discussion)

**User's proposal:** Remove role column entirely. Use is_superuser flag for admins. Presence in tenant_memberships = manager. No explicit role anywhere.

| Option | Description | Selected |
|--------|-------------|----------|
| All members equal — no role column | Membership = full manager access. No tiers. | ✓ |
| Keep role on membership | membership.role allows permission tiers. | |
| Boolean flags on membership | Fine-grained per-permission flags. | |

**User's choice:** All members equal — no role column
**Notes:** "I just want to add something: document this properly, as by seeing just the code its not very obvious that the system flows it properly."

## Password Hashing

**User raised:** Switch from scrypt (current) to argon2 with salt.

**Resolution:** Install `argon2` npm package. Update `config/hash.ts` to argon2 as default. Update `withAuthFinder` in User model. Argon2 handles salting automatically.
