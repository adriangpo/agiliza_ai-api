  <action>
    Step 1 — Ensure lefthook and commitlint packages are installed:
    ```
    npm install -D lefthook@^2.0.0 @commitlint/cli@^20.0.0 @commitlint/config-conventional@^20.0.0
    ```

    Step 2 — Create `lefthook.yml` (D-19: binary, no Node.js runtime at hook execution, D-20: pre-commit, D-21: commit-msg):
    ```yaml
    # lefthook.yml
    # INFRA-02 — D-19, D-20, D-21
    # Pre-commit: lint staged TypeScript files, format, and type-check.
    # commit-msg: enforce Conventional Commits format.
    pre-commit:
      parallel: false
      commands:
        lint:
          glob: "*.{ts,js}"
          run: npx eslint --max-warnings 0 {staged_files}
        format:
          glob: "*.{ts,js}"
          run: npx prettier --write {staged_files} && git add {staged_files}
        typecheck:
          run: npx tsc --noEmit

    commit-msg:
      commands:
        commitlint:
          run: npx commitlint --edit {1}
    ```

    Step 3 — Create `.commitlintrc.json` (D-21: Conventional Commits, allowed types):
    ```json
    {
      "extends": ["@commitlint/config-conventional"],
      "rules": {
        "type-enum": [
          2,
          "always",
          ["feat", "fix", "chore", "docs", "test", "refactor", "perf", "ci"]
        ]
      }
    }
    ```

    Step 4 — Register hooks with Lefthook (writes to .git/hooks/):
    ```
    npx lefthook install
    ```

    Step 5 — Add a `prepare` script to `package.json` so Lefthook installs automatically on `npm install`:
    Add to `package.json` `scripts`:
    ```json
    "prepare": "lefthook install"
    ```

    Step 6 — Verify the commit-msg hook would reject an invalid message:
    ```
    echo "bad commit message" | npx commitlint
    ```
    This should exit non-zero (reject). A valid message `"feat: add scaffold"` should exit 0.

    Do NOT use `simple-git-hooks`, `lint-staged`, or `husky` — they are banned per the research (D-19 explicitly locks Lefthook).
  </action>
