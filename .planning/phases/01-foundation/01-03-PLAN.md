    ```yaml
    # docker-compose.yml
    # D-15: Local dev services. Single source of truth for service config.
    # postgis/postgis:17-3.5 = PostgreSQL 17 with PostGIS 3.5 (matches CI)
    services:
      postgres:
        image: postgis/postgis:17-3.5
        environment:
          POSTGRES_USER: agiliza_ai
          POSTGRES_PASSWORD: password
          POSTGRES_DB: agiliza_ai
        ports:

        volumes:
          - postgres_data:/var/lib/postgresql/data
        healthcheck:

          interval: 10s
          timeout: 5s
          retries: 5

      redis:
        image: redis:7-alpine
        ports:
          - "6379:6379"
        healthcheck:
          test: ["CMD", "redis-cli", "ping"]
          interval: 10s
          timeout: 3s
          retries: 5

    volumes:
      postgres_data:
    ```

    Create `Makefile` at project root. All developer commands go through make targets. Never bypass with raw commands.

    Required targets from D-17 + make typecheck + make dev + make build:

    ```makefile
    # Makefile
    # D-16: Makefile is law. All commands go through make targets.
    # Agents (Claude, CI, docs) MUST use make targets — never raw commands.
    # If a target fails, fix the Makefile; do not bypass it.

    .PHONY: up down dev build test test-watch lint lint-fix typecheck migrate migrate-fresh

    # ── Docker Services ──────────────────────────────────────────────────────────
    up:
    	docker compose up -d

    down:
    	docker compose down

    # ── Development ──────────────────────────────────────────────────────────────
    dev:
    	node ace serve --watch

    build:
    	node ace build

    # ── Testing ──────────────────────────────────────────────────────────────────
    # D-18: NODE_ENV=test routes to the test database (agiliza_ai_test)
    test:
    	NODE_ENV=test node ace test

    test-watch:
    	NODE_ENV=test node ace test --watch

    # ── Code Quality ─────────────────────────────────────────────────────────────
    # D-22: --max-warnings 0 always. Zero tolerance for warnings.
    lint:
    	npx eslint . --max-warnings 0

    lint-fix:
    	npx eslint . --fix --max-warnings 0

    typecheck:
    	npx tsc --noEmit

    # ── Database ─────────────────────────────────────────────────────────────────
    # D-07: Migrations run as 'migrator' role (DDL owner). Never run as 'app'.
    migrate:
    	node ace migration:run

    migrate-fresh:
    	node ace migration:fresh --seed
    ```

    CRITICAL: Makefile indentation uses TABS (not spaces). Targets indented with spaces will silently fail. Use a tab character on every command line.

    Verify the Makefile syntax immediately after creating it:
    ```
    make --dry-run up
    make --dry-run lint
    ```

    Note: `make test:watch` was specified in D-17 but Makefile targets cannot contain colons. Use `make test-watch` as the target name and document this in a comment.
    Note: `make migrate:fresh` was specified in D-17 but Makefile targets cannot contain colons. Use `make migrate-fresh` as the target name (per D-17 intent, corrected for Makefile syntax).
  </action>
