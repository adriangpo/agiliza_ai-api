# Phase 01 Plan 02: Lint, Format, and Git Hooks Summary

**One-liner:** ESLint v10 flat config with AdonisJS preset, Prettier with AdonisJS preset, and Lefthook hooks enforcing lint + Conventional Commits on every commit.

## What Was Built

### Task 1: ESLint v10 flat config + Prettier (commit: 6cba482)

Created `eslint.config.ts` replacing the scaffold's `eslint.config.js`. The TypeScript flat config uses `@adonisjs/eslint-config`'s `configApp()` factory (ESLint v10 flat config style) and explicitly ignores `database/schema.ts` (auto-generated by Lucid v22) and the `build/` directory.

Created `.prettierrc` as a standalone file (the string `"@adonisjs/prettier-config"`) following the Prettier "extends" pattern. Removed the `prettier` key from `package.json` to avoid duplicate config resolution.

Updated `.prettierignore` to exclude `.planning/` (documentation files), `CLAUDE.md`, and `database/schema.ts` from Prettier formatting checks.

Added scripts to `package.json`:
- `format:check` — `prettier --check .`
- `prepare` — `lefthook install` (auto-installs hooks on `npm install`)
- Fixed `lint:fix` flag order to `--fix --max-warnings 0` per plan spec

Auto-fixed two Prettier violations in scaffold-generated files:
- `adonisrc.ts` line 53: preloads array formatting
- `tests/bootstrap.ts` line 16: plugins array formatting

### Task 2: Lefthook + commitlint hooks (commits: 6bc5635, 9cd85c9)

Replaced the example-only `lefthook.yml` with actual hook configuration implementing D-19, D-20, D-21:

**pre-commit** (sequential, not parallel):
- `lint`: runs `npx eslint --max-warnings 0 {staged_files}` on `*.{ts,js}`
- `format`: runs `npx prettier --write {staged_files} && git add {staged_files}` on `*.{ts,js}`
- `typecheck`: runs `npx tsc --noEmit`

**commit-msg**:
- `commitlint`: runs `npx commitlint --edit {1}`

Created `.commitlintrc.json` extending `@commitlint/config-conventional` with a custom `type-enum` rule allowing: `feat`, `fix`, `chore`, `docs`, `test`, `refactor`, `perf`, `ci`.

Installed hooks into `.git/hooks/` via `lefthook install --force` (force required in worktree context).
