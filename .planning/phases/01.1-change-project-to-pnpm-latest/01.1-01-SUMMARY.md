## Decisions Made

- Used `pnpm import` (not plain `pnpm install`) to avoid upgrading packages within their semver ranges. A direct `pnpm install` would have drifted `@adonisjs/core` from `7.1.1` to `7.2.0` and `@adonisjs/assembler` from `8.2.0` to `8.3.0`.
- Did not add hoisting config. Research confirmed pnpm 10's default isolated layout works for this repo without `.npmrc` or `pnpm-workspace.yaml`. Build-script warnings for `@swc/core` and `lefthook` appeared but did not break install.
- Deleted `package-lock.json` only after `pnpm install --frozen-lockfile` exited 0 and all sentinel versions were grep-verified in `pnpm-lock.yaml`.
